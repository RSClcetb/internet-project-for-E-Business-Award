Word Processors

There are a lot of options when it comes to word processors, Word has both on and offline iterations that integrate with a lot of software, there is also google docs and open source options like libreoffice.

There are a number of reasons for using a word processor when developing websites but the main one is that functionality like spellchecking and formatting are features that are much harder to implement easily in a coding environment.

  • Microsoft Word
  • Google Docs
  • Libre Office

Desktop publishing programs

Originally these software packages were used exclusively for print content, compiling the layout of a variety of content and formatting it correctly. Newspapers, book publishers and marketers still use desktop publishing software for this purpose.

as webpages developed into being more capable this software now has a place in web development as it provides the abilty to mix multimedia and typographic elements effectively.

the most popular desktop publishing programmes are Adobe inDesign and microsoft publisher

  • Adobe Indesign
  • Microsoft Publisher

Webpage Builders and HTML editors

There are now a lot of options when it comes to building a website, from coding yourself in a package like dreamweaver or notepad ++ to what you see is what you get interfaces (wysiwyg) offered at a premium from services like squarespace and wix.

there are pros and cons to both of these approaches, coding allows for much more freedom to customise but is difficult for beginners and can be time consuming.

WYSIWYG solutions tend to be beginner friendly and usually the results are quite good but the customisation is limited and tend to come at a premium.

  • notepad++
  • Wix
  • Adobe Dreamweaver
